1. What are the kinds of problems we see in nature?

Ans:
Iteration: Repeating tasks like summing a list of numbers or calculating compound interest.
Recursion: Scenarios like generating the Fibonacci sequence or solving the Tower of Hanoi involve recursive calls.
Backtracking: Solving problems like mazes, n-queens, or generating permutations using backtracking.

2. What is space and time efficiency? Why are they important? Explain different class of problems and orders of growth.

Ans:
Time Efficiency measures the computational time required to execute an algorithm. Space Efficiency measures the memory (storage) required. A good algorithm should strike a balance between time and space efficiency.
Big O Notation (Complexity Classes):

  • O(1) - Constant Time: Fixed runtime (e.g., simple arithmetic).
  • O(log n) - Logarithmic Time: Efficient for searching (e.g., binary search).
  • O(n) - Linear Time: Traversing an array (e.g., linear search).
  • O(n log n) - Linearithmic Time: Sorting (e.g., merge sort).
  • O(n²) - Quadratic Time: Nested loops (e.g., bubble sort).
  • O(2ⁿ) - Exponential Time: Recursive solutions like Fibonacci.
  • O(n!) - Factorial Time: Inefficient algorithms (e.g., brute-force).

3. Take away from different design principles from chapter 2:

Modularity: Breaking down complex problems into smaller, manageable tasks.
Abstraction: Hiding implementation details to focus on solving the problem.
Efficiency: Optimizing time and space complexities.
Reusability: Using solutions like dynamic programming for different domains.

4. The hierarchical data and how different tree data structures solve and optimize problems:

General Tree: Represents parent-child relationships but lacks balance.
Binary Search Tree (BST): Efficient searching with balanced properties (O(log n) on average).
AVL Tree: Self-balancing for consistent O(log n) operations.
Heap: Used in priority queues, ensuring fast insert/delete operations.
Trie: Optimized for prefix-based searches like autocomplete.

5. The need for array query algorithms and their implications:

Segment Trees: Efficient range queries like sum or min operations in O(log n).
Fenwick Tree (BIT): Prefix sums and updates in O(log n).
Sparse Table: Static range queries in O(1) after preprocessing (O(n log n)).
Applications: Used in real-time systems, databases, stock analysis, and big data.
