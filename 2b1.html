Portfolio Case: Emergency Fire Detection and Management

Overview

Sub-task: Fire Alert System

SDG Goal: 11 - Sustainable Cities and Communities

Target: 11.5 - Reduce the number of deaths and affected individuals by disasters.

Indicator: 11.5.1 - Number of deaths, missing persons, and directly affected persons attributed to disasters.

Business Case

The fire department of the city monitors emergency fire incidents using a combination of IoT devices, data structures, and algorithms to ensure quick detection and response. The system processes real-time data from sensors and determines the shortest path for fire engines to reach the incident site.

Data Representation

  • Sensor Nodes: Represented as graph vertices.
  • Connections Between Nodes: Represented as graph edges with weights corresponding to distance or time required.
  • Fire Station Locations: Marked as priority vertices for routing.

Real-World Data Used

  • Sensor Data: Locations of IoT sensors in MB format.
  • Population Density: Used to prioritize response based on risk level.
  • Historical Fire Incident Data: Helps train prediction models.

Algorithms and Techniques Used

  • Dijkstra's Algorithm: Determine the optimal path for fire engines.
  • Heap Data Structure: Efficient priority queue implementation in Dijkstra's algorithm.
  • A* Algorithm: Enhanced routing with heuristic information.
  • Kruskalâ€™s Algorithm: Construct the optimal network of fire response units.
  • Binary Search Tree (BST): Efficient storage and retrieval of historical incident data.

Process Flow

  1. Detection: Sensors detect abnormal temperature or smoke levels.
  2. Data Aggregation: Sensor data is aggregated and processed.
  3. Response Planning:
    • Use Dijkstra's algorithm to find the quickest route from the nearest fire station to the fire location.
    • Calculate risk levels based on nearby population density using weighted factors.
  4. Execution: Fire engines are dispatched, and routes are continuously updated based on real-time traffic data.
