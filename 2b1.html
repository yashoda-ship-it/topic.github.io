Emergency Fire Detection and Management

Overview

Objective: Create a robust system for detecting fires using IoT devices and optimizing emergency response times through Dynamic Programming and Memoization.

SDG Goal: Sustainable Cities and Communities (Goal 11).

Target: Reduce disaster-related deaths and affected individuals.

Data Representation

The system is modeled as a graph, where:

  • Vertices: IoT sensor nodes or fire stations, represented using Kruskal's Algorithm for network optimization.
  • Edges: Routes with weights indicating travel time or distance, optimized through Edge Relaxation techniques.
  • Population Density: Added as a factor to prioritize response using Priority Queues and Greedy Techniques.

Algorithms

Shortest Path: Dijkstra's Algorithm

Finds the optimal route for fire engines from the nearest station to the fire location using Edge Relaxation.

Network Optimization: Kruskal's Algorithm

Constructs the minimal spanning tree to reduce overall sensor connection costs.

Dynamic Routing: A* Algorithm

Incorporates heuristics for real-time pathfinding based on traffic data using Lazy Propagation and Sliding Window.

Data Compression: Huffman Coding

Compresses historical fire data to reduce storage requirements.

Process Flow

The emergency fire detection system follows these steps:

  1. IoT sensors detect abnormal conditions (temperature, smoke) using Bit Manipulations.
  2. Data is aggregated and sent to the central system via Union-Find data structures for connectivity analysis.
  3. Algorithms calculate the quickest route for fire response using Shortest Path Trees.
  4. Fire engines are dispatched with dynamic route updates via Lazy Propagation.

Optimization Techniques

  • Real-Time Updates: Routes are dynamically adjusted using Sliding Window and Dynamic Programming.
  • Risk Prioritization: High-risk zones are prioritized based on population density using Greedy Techniques.
  • Storage Optimization: Huffman Coding minimizes storage overhead for historical fire data.
