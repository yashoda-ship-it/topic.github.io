Sub-task: Fire Alert System

SDG Goal: 11 - Sustainable Cities and Communities

Indicator: 11.5.1 - Number of deaths, missing persons, and directly affected persons attributed to disasters.

Business Case

The fire department monitors emergency fire incidents using IoT devices, data structures, and algorithms to ensure quick detection and response. The system processes real-time data from sensors and determines the shortest path for fire engines to reach the incident site. Population density and historical incident data help in prioritizing high-risk zones.

Data Representation

Data is represented using graphs where:

  • Sensor Nodes: Represented as vertices in the graph.
  • Connections: Represented as edges with weights corresponding to distance or time required.
  • Fire Station Locations: Marked as priority vertices for routing fire engines.

Algorithms Used

Key algorithms for fire detection and management:

  • Dijkstra's Algorithm: Used to find the shortest path between fire stations and the fire incident location.
  • A* Algorithm: Enhances Dijkstra by adding heuristics for faster routing.
  • Kruskal's Algorithm: Builds the optimal sensor network by minimizing the connection costs.
  • Binary Search Tree (BST): Efficiently stores and retrieves historical fire incident data.
  • Huffman Coding: Compresses historical data for efficient storage.

Dijkstra's Algorithm Code Example

def dijkstra(graph, start):
    import heapq
    pq = []  # Priority queue
    heapq.heappush(pq, (0, start))
    distances = {node: float('inf') for node in graph}
    distances[start] = 0

    while pq:
        current_distance, current_node = heapq.heappop(pq)

        for neighbor, weight in graph[current_node]:
            distance = current_distance + weight
            if distance < distances[neighbor]:
                distances[neighbor] = distance
                heapq.heappush(pq, (distance, neighbor))

    return distances

Process Flow

  1. Detection: Sensors detect abnormal temperature or smoke levels.
  2. Data Aggregation: Sensor data is aggregated and processed in real-time.
  3. Response Planning:
    • Use Dijkstra's algorithm to find the quickest route to the fire site.
    • Calculate risk levels based on nearby population density.
  4. Execution: Fire engines are dispatched, and routes are updated dynamically.

Optimization Techniques

  • Dynamic Routing: BFS/DFS algorithms dynamically adjust routes based on traffic data.
  • Data Compression: Huffman coding reduces storage requirements for historical data.
  • Heuristics: A* Algorithm improves response time by estimating target proximity.
