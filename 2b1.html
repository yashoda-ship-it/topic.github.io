Emergency Fire Detection and Management

SDG Mapping

Objective: Create a robust system for detecting fires using IoT devices and optimizing emergency response times through Dynamic Programming and Memoization.

SDG Goal: Sustainable Cities and Communities (Goal 11).

Target: Reduce disaster-related deaths and affected individuals (Target 11.5).

Indicator: Number of deaths, missing persons, and persons affected by disaster, per 100,000 people (Indicator 11.5.1).

Data Representation

The system is modeled as a graph, where:

  • Vertices: IoT sensor nodes or fire stations, represented using Kruskal's Algorithm for network optimization.
  • Edges: Routes with weights indicating travel time or distance, optimized through Edge Relaxation techniques.
  • Population Density: Added as a factor to prioritize response using Priority Queues and Greedy Techniques.

Algorithms

Shortest Path: Dijkstra's Algorithm

Finds the optimal route for fire engines from the nearest station to the fire location using Edge Relaxation. Efficiency: Time complexity of O(E log V) where E is the number of edges and V is the number of vertices.

  • Dijkstra's Algorithm Code

Network Optimization: Kruskal's Algorithm

Constructs the minimal spanning tree to reduce overall sensor connection costs. Efficiency: Time complexity of O(E log V), where E is the number of edges and V is the number of vertices.

  • Kruskal's Algorithm Code

Priority Queue: Min-Heap

The Min-Heap is used to manage the priority of fire engines dispatched to locations based on urgency. Efficiency: Time complexity of O(log n) for both insertion and removal of elements.

  • Min-Heap Code

Union-Find: Disjoint-Set Union

The Union-Find data structure is used for efficiently checking connectivity between different fire stations and fire locations in the network. Efficiency: Time complexity of O(α(n)), where α(n) is the inverse Ackermann function.

  • Union-Find Code

Segment Tree

Used for querying ranges of data (e.g., population density in different areas) efficiently. Efficiency: Time complexity for both updates and queries is O(log n).

  • Segment Tree Code

Trie

Used for efficiently searching and storing fire-related keywords (such as location names or types of incidents). Efficiency: Time complexity for insertion and searching is O(k), where k is the length of the string.
